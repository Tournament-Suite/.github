# Security Policy

## Reporting a Vulnerability

Tournament Suite takes security seriously. If you discover a security vulnerability in our platform, API, or any publicly accessible service, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### How to Report

Email us at **security@tournamentsuite.com** with:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested remediation (optional)

We will acknowledge your report within **48 hours** and provide a resolution timeline within **7 days**.

## Scope

The following are in scope for security reports:

- [tournamentsuite.com](https://tournamentsuite.com) and all subdomains
- [arena.gamertd.com](https://arena.gamertd.com) (live demo)
- The Tournament Suite REST and WebSocket API
- Authentication and authorization flows
- Player data handling and privacy

## Out of Scope

- Social engineering attacks
- Denial of service (DoS/DDoS)
- Issues in third-party services we depend on (report those upstream)
- Vulnerabilities requiring physical access to infrastructure

## Responsible Disclosure

We ask that you:

- Give us reasonable time to investigate and fix before public disclosure
- Avoid accessing, modifying, or deleting data that does not belong to you
- Act in good faith

We commit to not pursuing legal action against researchers who follow these guidelines and report in good faith.

## Supported Versions

As a pre-production platform, all active environments receive security fixes. We do not maintain legacy versions.
